常见报错解决方法

npm WARN checkPermissions Missing write access to…

若出现此类报错，删除hexo目录下node_modules的全部文件，之后使用npm install 重新安装,别忘了

npm WARN invalid config registry=””

这个问题困扰了我很久，虽然在部署的时候是正确的，但每次在hexo g和hexo d的时候都会报出两行错误的npm注册表，但今后可能会出bug。

解决方法：卸载Node.js，将C:\Users\xxx\AppData\Roaming\npm和C:\Users\xxx\AppData\Roaming\npm-chache删除(xxx为您的电脑用户名)，重新安装Node.js，在cmd命令中输入npm -v和node -v，只在一行中显示正确版本即可。
